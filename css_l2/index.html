TPSIT CSS — Capitolo: Selectors

Lezione 2 — Raggruppare selettori e dichiarazioni (grouping) + selettore universale *

Oggi fate un salto di qualità: scrivete CSS più pulito, più breve e più facile da mantenere. Attenzione alla virgola: a volte cambia completamente il significato di una regola.

⏱️ 1 ora • Classe: 4ª IT • Modalità: laboratorio

Obiettivi (oggi)

  • Usare la virgola per applicare lo stesso stile a più selettori (grouping selectors).
  • Scrivere blocchi CSS compatti raggruppando più dichiarazioni nello stesso { ... }.
  • Conoscere il selettore universale * e quando è utile (e quando no).
Attenzione: h2, p (con virgola) e h2 p (senza virgola) NON sono la stessa cosa. Oggi dovete saperlo spiegare.

Prerequisiti

  • Saper scrivere una regola CSS corretta (Lezione 1).
  • Capire cosa sono h1, p, classi (.intro) e ID (#fine) a livello base.
  • DevTools pronti: F12 per controllare se la regola è applicata.
Consegna Moodle (fine lezione): caricate il vostro style.css (o incollate il CSS) + 2 righe di commento: “cosa ho raggruppato e perché”.

Warm-up (10 minuti)

Guardate queste due regole e rispondete: a quali elementi si applica ciascuna? (Scrivete una frase in italiano, non solo “a tutto”.)

Regola A (con virgola)

h2, p {
  color: #666;
}

Regola B (senza virgola)

h2 p {
  color: #666;
}
Risposta attesa (idea): la virgola crea una “lista” di selettori separati. Senza virgola, lo spazio cambia tutto: significa “p dentro h2” (e spesso non esiste).

1) Grouping selectors: una regola, più bersagli

Quando più elementi devono avere lo stesso stile, evitate di ripetere il blocco.

❌ Ripetitivo

h2 { color: #444; }
p  { color: #444; }
li { color: #444; }

✅ Raggruppato

h2, p, li {
  color: #444;
}
Regola pratica: raggruppate solo ciò che ha senso “stare insieme”. Se poi p cambia, dovete poterlo separare facilmente.

2) Raggruppare dichiarazioni: un blocco ben scritto

Dentro le graffe potete (e dovete) mettere più dichiarazioni, una per riga. È il modo “normale” di scrivere CSS. 

.card {
  padding: 16px;
  border: 1px solid #ddd;
  border-radius: 12px;
  background: #fff;
}
Errore tipico: dimenticare il ; e far “saltare” la lettura del resto. Se qualcosa non funziona, cercate prima quello.

3) Il selettore universale *: potente ma da usare con criterio

* seleziona tutti gli elementi. È utile per piccoli reset, per box-sizing, o per debugging. Ma può anche rendere il CSS più “aggressivo” del necessario.

Esempio utile: box-sizing globale

* {
  box-sizing: border-box;
}

Così padding e bordi non “allargano” in modo strano gli elementi.

Esempio da evitare: stile “a martello”

* {
  color: red;
}

Funziona, ma poi dovete combattere per sovrascrivere ovunque.

Consiglio: usate * per poche cose “di base” (reset/box-model). Per il resto, selettori più specifici = meno problemi.

Attività pratica (25 minuti) — “CSS da verboso a pulito”

Scaricate/ricopiate questi due file in una cartella. Il CSS funziona, ma è ripetitivo. Il vostro compito è ridurre righe e ripetizioni usando: grouping selectors, blocchi ordinati, e (se serve) *.

File 1 — index.html

<!doctype html>
<html lang="it">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Lezione 2 — Grouping</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <main class="wrap">
    <h1>Grouping selectors</h1>
    <p class="lead">Obiettivo: meno ripetizioni, stesso risultato.</p>

    <section class="card">
      <h2>Titolo sezione</h2>
      <p>Testo di esempio.</p>
      <ul>
        <li>Voce 1</li>
        <li>Voce 2</li>
      </ul>
      <a class="btn" href="#">Azione</a>
    </section>
  </main>
</body>
</html>

File 2 — style.css (da rifattorizzare)

body {
  font-family: system-ui;
  background: #f6f6f6;
  color: #222;
}

.wrap {
  max-width: 900px;
  margin: 0 auto;
  padding: 18px;
}

h1 {
  color: #222;
  letter-spacing: 0.5px;
}

h2 {
  color: #222;
  letter-spacing: 0.5px;
}

p {
  color: #222;
  line-height: 1.6;
}

li {
  color: #222;
  line-height: 1.6;
}

.card {
  background: #fff;
  border: 1px solid #ddd;
  border-radius: 12px;
  padding: 16px;
}

.btn {
  display: inline-block;
  margin-top: 10px;
  padding: 10px 12px;
  background: #222;
  color: #fff;
  text-decoration: none;
  border-radius: 10px;
}
Vincoli (obbligatori):
  • Usate almeno 2 raggruppamenti con la virgola (es: h1, h2).
  • Non cambiate l’HTML.
  • Il risultato visivo deve rimanere “praticamente uguale”.
Bonus (facoltativo): aggiungete * { box-sizing: border-box; } e motivate in 1 riga.

Consegna Moodle (10 minuti finali)

  1. Salvate il vostro style.css definitivo.
  2. Caricatelo su Moodle (compito “Lezione 2 — Grouping”).
  3. Nel testo della consegna scrivete 2 righe:
    • Quali selettori avete raggruppato?
    • Perché era sensato raggrupparli?
Checklist rapida prima di consegnare:
  • Ci sono almeno 2 regole con , (grouping)?
  • Avete evitato ripetizioni inutili?
  • Il CSS è leggibile (una dichiarazione per riga, indentazione)?
