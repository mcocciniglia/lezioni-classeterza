Installare VS Code e vedere l’anteprima delle pagine HTML

Guida da svolgere a casa. Obiettivo: aprire un file .html e vederlo nel browser con un preview rapido.

Modalita: a casa • Strumenti: PC + connessione Internet

Cosa faremo

  • Installare Visual Studio Code (versione normale) oppure preparare la versione portable.
  • Aprire una cartella di lavoro e creare un file index.html.
  • Installare e usare un’estensione per il preview delle pagine (consigliata: Live Server).
Link utili (ufficiali)
  • Download VS Code: https://code.visualstudio.com/download
  • Guida ufficiale “Portable Mode”: https://code.visualstudio.com/docs/editor/portable

1) Installazione “normale” (consigliata): Windows

  1. Scaricate VS Code dal sito ufficiale

    Andate su code.visualstudio.com/download e nella sezione Windows scegliete: User Installer (x64) (nella maggior parte dei PC).

    Se non sapete quale scegliere

    Di solito va bene User Installer x64. Se avete un PC ARM (raro) scegliete Arm64. Se non avete i permessi di amministratore, User Installer e spesso la scelta piu semplice.

  2. Eseguite l’installer

    Aprite il file scaricato (di solito si chiama VSCodeUserSetup-....exe) e seguite la procedura guidata.

    Consigli durante l’installazione
    • Se compare l’opzione Add to PATH, potete attivarla (utile, ma non obbligatoria).
    • Se compare l’opzione per aggiungere “Apri con Code” nel menu tasto destro, potete attivarla (comoda per aprire cartelle).

  3. Avvio iniziale

    Alla fine, avviate Visual Studio Code dal menu Start.

Se usate Mac o Linux

La logica e molto simile: si scarica dal sito ufficiale e si installa il pacchetto adatto. Partite sempre dalla pagina: https://code.visualstudio.com/download

2) Alternativa: VS Code “portable” (senza installare)

“Portable” significa che VS Code gira da una cartella (anche su chiavetta) e puo tenere impostazioni ed estensioni nella stessa cartella, senza installare nulla nel sistema.

Quando conviene portable?
  • Non avete permessi di installazione sul PC.
  • Volete portare VS Code su una chiavetta e usarlo su piu PC.
  • Volete tenere tutto “in una cartella” (programma + dati + estensioni).

  1. Scaricate la versione .zip

    Dal sito di download ufficiale scegliete .zip (x64) invece dell’installer: https://code.visualstudio.com/download

  2. Estraete lo ZIP in una cartella vostra

    Esempio: D:\Programmi\VSCodePortable\ oppure una cartella su Desktop/Documenti. Dentro troverete l’eseguibile Code.exe.

  3. Attivate il Portable Mode creando la cartella data

    Nella stessa cartella dove c’e Code.exe, create una cartella chiamata esattamente: data. Guida ufficiale: https://code.visualstudio.com/docs/editor/portable

    Esempio struttura
    VSCodePortable\
  Code.exe
  data\
    user-data\
    extensions\
    Cartella data = impostazioni ed estensioni “portabili”.

  4. Avviate VS Code

    Fate doppio clic su Code.exe. Se la cartella data esiste, VS Code usera la modalita portable.

3) Primo progetto: create una cartella e un file HTML

  1. Create una cartella di lavoro

    Sul PC create una cartella chiamata per esempio sito_prova.

  2. Aprite la cartella in VS Code

    In VS Code: File → Open Folder... e scegliete la cartella sito_prova.

    Perche “Open Folder” e importante?

    Le estensioni di preview lavorano meglio se aprite una cartella progetto, non un singolo file.

  3. Create index.html

    Nel pannello a sinistra (Explorer), cliccate su New File e chiamatelo index.html.

    Incollate questo HTML minimo:

    <!doctype html>
<html lang="it">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Prova preview</title>
</head>
<body>
  <h1>Ciao, preview!</h1>
  <p>Se leggi questo nel browser, funziona.</p>
</body>
</html>

    Salvate con Ctrl + S.

4) Attivare il “preview” delle pagine: due strade (sceglietene una)

  • Server locale con auto-reload (consigliato): apre la pagina nel browser e aggiorna quando salvate.
  • Preview integrato (dentro VS Code): utile se non volete uscire dall’editor.

Opzione A (consigliata): Live Server (Ritwick Dey)

Live Server avvia un piccolo server sul vostro PC e apre la pagina nel browser. Quando salvate, la pagina si aggiorna.

Link Marketplace

https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer

  1. Aprite il pannello Estensioni

    Cliccate l’icona Extensions oppure premete Ctrl + Shift + X.

  2. Cercate e installate “Live Server”

    Nella ricerca scrivete: Live Server e controllate che il publisher sia Ritwick Dey.

    In alternativa: Ctrl + P e scrivete: ext install ritwickdey.liveserver

  3. Avviate il server

    • Aprite index.html e cliccate Go Live (se appare), oppure
    • Tasto destro sul file → Open with Live Server, oppure
    • Ctrl + Shift + P → Live Server: Open with Live Server.

  4. Test

    Cambiate una parola in index.html, salvate con Ctrl + S e guardate il browser: dovrebbe aggiornarsi.

Se non parte o si blocca
  • Controllate di aver aperto una cartella (Open Folder), non solo il file.
  • Provate a riavviare VS Code.
  • Se vi da errore di “porta occupata”, chiudete altri server o cambiate porta nelle impostazioni di Live Server.

Opzione B: Live Preview (Microsoft)

Esiste anche un’estensione Microsoft chiamata Live Preview. Funziona in modo simile (server locale + preview).

Link Marketplace

https://marketplace.visualstudio.com/items?itemName=ms-vscode.live-server

  1. Installazione

    Extensions → cercate Live Preview (publisher Microsoft), oppure: Ctrl + P → ext install ms-vscode.live-server

  2. Apertura preview

    Aprite index.html, poi Ctrl + Shift + P e cercate un comando tipo Live Preview / Start Server / Show Preview.

Se non trovate i comandi

Alcune estensioni aggiungono comandi diversi a seconda della versione di VS Code. In quel caso usate l’opzione A (Live Server), che e quella piu semplice e diffusa.

Checklist finale (autovalutazione)

  • [ ] VS Code si avvia (installato o portable).
  • [ ] La cartella sito_prova e aperta in VS Code.
  • [ ] Il file index.html e stato creato e salvato.
  • [ ] E stata installata un’estensione di preview (Live Server consigliata).
  • [ ] La pagina si vede nel browser e si aggiorna quando salvate.
