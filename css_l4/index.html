Lezione 4 — Classi e ID: quando usarli e perché

Oggi lavorate “da sviluppatori”: non solo scrivere CSS, ma anche scegliere un HTML che si lasci stilare bene. Imparate a usare classi per gruppi e ID per elementi unici.

⏱️ 1 ora • Classe: 4ª IT • Modalità: laboratorio

Obiettivi (oggi)

  • Usare correttamente .classe e #id nei selettori.
  • Capire la regola pratica: classe = ripetibile, ID = unico.
  • Scrivere CSS riutilizzabile: una classe “base” + piccole varianti.
  • Introduzione “soft” alla specificità: spesso #id vince su .classe.
Errore tipico: usare lo stesso id su più elementi. In HTML l’ID deve essere unico nella pagina.

Prerequisiti

  • Lezione 1: sintassi delle regole CSS.
  • Lezione 2: raggruppamento con virgola.
  • Lezione 3: combinatori (spazio, >, +, ~).
Consegna Moodle (fine lezione): caricate una cartella ZIP con index.html + style.css (oppure caricate i 2 file separati se Moodle lo consente).

Warm-up (10 minuti) — “classe o ID?”

Per ciascun caso scrivete: “userei una classe” oppure “userei un ID”, e perché.

  1. Il bottone “Contattaci” che compare in 3 sezioni diverse.
  2. Il blocco “Cookie banner” (uno solo in pagina).
  3. Tutti i paragrafi di introduzione (stesso stile in più pagine).
  4. Il link “Torna su” che punta a un punto preciso della pagina.
Regola pratica: se uno stile deve essere riusato, pensate prima alla classe. L’ID è ottimo per “pezzi unici” e per ancore.

1) Classi: “etichette” riutilizzabili

Una classe si assegna con class="nome" e si seleziona con .nome. Potete metterla su quanti elementi volete. 


<a class="btn" href="#">Azione</a>
<a class="btn" href="#">Scarica</a>
.btn {
  display: inline-block;
  padding: 10px 12px;
  border-radius: 10px;
  text-decoration: none;
}
Classi multiple: potete assegnare più classi allo stesso elemento: class="btn btn-primary".

2) ID: elemento unico + ancore

Un ID si assegna con id="nome" e si seleziona con #nome. Deve comparire una sola volta nella pagina. 

<h2 id="contatti">Contatti</h2>
<a href="#contatti">Vai ai contatti</a>
#contatti {
  scroll-margin-top: 20px;
}
Non usate l’ID per “tutto”. Se lo riusate, state cercando una classe.

3) Specificità (introduzione pratica)

Quando due regole provano a stilare lo stesso elemento, il browser decide “chi vince”. Senza fare formule: spesso #id è più forte di .classe. 

<a id="cta" class="btn" href="#">Iscriviti</a>
.btn { background: #222; color: #fff; }
#cta  { background: #0b5; }
Idea chiave: usate questa “forza” con giudizio. Meglio costruire CSS con classi ben pensate, e usare l’ID solo quando serve davvero.

Attività pratica (25 minuti) — “HTML che si lascia stilare”

Create una cartella con questi file. Obiettivo: aggiungere classi sensate, usare un ID unico per l’ancora, e scrivere un CSS pulito e riutilizzabile.

File 1 — index.html (da completare)

<!doctype html>
<html lang="it">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Lezione 4 — Classi e ID</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <header>
    <h1>Mini pagina</h1>
    <p>Scegliete classi/ID per rendere il CSS riutilizzabile.</p>
    <nav>
      <a href="#sezione">Vai alla sezione</a>
      <a href="#fine">Vai in fondo</a>
    </nav>
  </header>

  <main>
    <section>
      <h2 id="sezione">Sezione</h2>
      <p>Qui c’è un testo normale.</p>

      <a href="#">Azione 1</a>
      <a href="#">Azione 2</a>
      <a href="#">Azione 3</a>
    </section>

    <section>
      <h2>Altra sezione</h2>
      <p>Altro testo.</p>

      <a href="#">Azione 4</a>
      <a href="#">Azione 5</a>
    </section>

    <footer>
      <p id="fine">Fine pagina.</p>
      <a href="#top">Torna su</a>
    </footer>
  </main>

</body>
</html>
Nota: c’è un problema: il link “Torna su” punta a #top ma nell’HTML non esiste. Sistemate usando un ID unico.

File 2 — style.css (da scrivere voi)

/* Compito:
1) Date una classe comune a tutti i link-azione (es: .btn) e stilateli come bottoni.
2) Create una variante (es: .btn-primary) per 1 solo bottone “importante”.
3) Stilate i contenitori sezione come “card” (bordo leggero + padding + raggio).
4) Rendete il menu (nav) leggibile: link senza sottolineatura, un po’ di spazio tra loro.
5) Usate 1 ID solo per l’ancora “top” e fate funzionare “Torna su”.

Vincoli:
- Niente ID ripetuti.
- Almeno 2 classi riutilizzabili (es: .card, .btn).
- CSS ordinato: una dichiarazione per riga, ; sempre.
*/
Suggerimento: prima aggiungete le classi in HTML (card, btn, ecc.), poi scrivete CSS. Se vi ritrovate a ripetere 3 volte la stessa regola, probabilmente vi manca una classe.

Consegna Moodle (10 minuti finali)

  1. Controllate che i link funzionino: “Vai alla sezione”, “Vai in fondo”, “Torna su”.
  2. Controllate che tutte le “azioni” abbiano lo stesso stile base (classe comune).
  3. Caricate index.html e style.css (ZIP o file singoli).
Checklist rapida:
  • Avete usato .btn per tutti i bottoni?
  • Avete usato .card per entrambe le sezioni?
  • Avete usato un solo #top e ora “Torna su” funziona?
  • Nessun ID ripetuto?

Exit ticket (Moodle) — 6 domande

  1. Differenza tra .classe e #id?
  2. Perché l’ID deve essere unico?
  3. Scrivete un esempio di elemento con due classi.
  4. Quando usate un ID invece di una classe?
  5. Chi “vince” di solito tra una regola con .classe e una con #id?
  6. Che vantaggio ha creare una classe base (.btn) e una variante (.btn-primary)?
Tempo: 7–9 minuti. Consegnate entro fine ora.
